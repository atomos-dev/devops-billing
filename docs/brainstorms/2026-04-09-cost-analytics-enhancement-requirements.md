# 成本分析页增强 — 汇总洞察 + 资源状态

## Problem Frame

成本分析页已有多维度图表和灵活的维度切换，但存在两个核心问题：
1. **缺少一眼可知的汇总指标** — 进入页面看不到总费用、环比变化等关键数字，需要用户在图表中自行估算
2. **只有费用维度，缺少资源维度** — 无法看到资源的使用状态（运行/停止），不知道是否有闲置资源在产生费用
3. **对比图与分布图功能重叠** — 两个图都是 breakdown by dimension，只是图表类型不同（饼图 vs 柱状图），浪费了一个位置

## Requirements

- R1. **汇总指标卡片区** — 在图表上方展示关键数字卡片：
  - 当月总费用
  - 月环比变化（金额和百分比，涨用红色/降用绿色）
  - 各 provider 的费用（跟随全局 provider 筛选）
- R2. **资源状态概览** — 用图表展示资源按状态（running/stopped/terminated）和 provider 的分布，替换现有的"费用对比"图（因为与分布图功能重叠）
- R3. **保留现有能力** — 趋势图（多维度切换）、分布图（多维度切换）、Top 10 资源排行保持不变

## Success Criteria

- 用户进入页面 1 秒内能看到当月总费用和环比变化方向
- 资源状态概览能明确显示各 provider 有多少运行/停止的资源
- 不增加额外的 API 请求（汇总指标可复用现有 trend API 数据计算）

## Scope Boundaries

- 不做复杂的异常检测算法（环比变化已足够识别明显波动）
- 不改变页面整体布局结构（仍是顶部筛选 + 下方内容区）
- 资源活跃度仅基于 status 字段判断，不引入 CPU/网络等监控指标

## Key Decisions

- **用资源状态图替换费用对比图**：费用对比图与分布图重叠，资源状态是一个全新的、互补的维度
- **环比计算在前端完成**：trend API 已返回连续月份数据，前端取最近两个月直接计算即可，无需新增后端接口
- **活跃度基于 status 字段**：简单直接，数据已有，不引入监控系统依赖
